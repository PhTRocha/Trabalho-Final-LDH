# Trabalho-Final-LDH
Trabalho de Modelagem de Sistemas Computacionais

👤 Integrante 1: Persistência de Dados e Modelagem EstruturalEste integrante será o responsável por preparar o "terreno" do jogo, garantindo que os dados existam e a estrutura do código siga a orientação a objetos.Modelagem UML: Desenvolver o Diagrama de Casos de Uso (descrevendo "Iniciar Jogo", "Fazer Adivinhação", etc.) e o Diagrama de Classes (modelando as classes principais como Jogo e Jogador).  Implementação em Java: Criar as classes base a partir do diagrama UML e implementar a persistência de dados lendo as palavras de um arquivo de texto.  Responsabilidade de Lógica: Garantir que o sistema consiga escolher uma palavra aleatória dessa lista predefinida.  

👤 Integrante 2: Lógica Principal (Motor do Jogo)Este integrante vai programar as regras do Jogo da Forca em si. Como a carga de programação lógica é maior, ele fica com um diagrama UML um pouco mais simples.Modelagem UML: Desenvolver o Diagrama de Atividades, modelando o fluxo de processos como "Iniciar Jogo" e "Fazer Adivinhação".  Implementação em Java (Adivinhação): Programar a lógica para receber as letras do jogador , verificar se a letra existe na palavra e diminuir as tentativas se a letra for incorreta.  Regras de Negócio: Fazer o sistema ignorar letras repetidas, pedindo uma nova adivinhação sem descontar tentativas.  Condição de Fim de Jogo: Criar a detecção de vitória (todas as letras adivinhadas) ou derrota (fim das tentativas).  

👤 Integrante 3: Interface CLI, Integração e DocumentaçãoEste integrante unirá o motor do jogo (feito pelo Integrante 2) com a base de dados (feita pelo Integrante 1), criando a interação com o usuário no terminal e finalizando os documentos de entrega.Modelagem UML: Desenvolver o Diagrama de Sequência, descrevendo a interação dos objetos ao fazer uma adivinhação.  Implementação em Java (Interface CLI): Criar a interface de linha de comando. Essa interface deve mostrar explicitamente: o tamanho da palavra, o nome do jogador, as letras adivinhadas corretamente, as letras incorretas tentadas e o número de tentativas restantes.  Fluxo do Jogo: Implementar a inicialização das tentativas na tela e a opção de iniciar um novo jogo ou sair ao final.  Documentação Final: Escrever o manual do usuário e organizar o arquivo ZIP da entrega com o código fonte completo, diagramas em PDF/PNG, manual, e arquivo com os nomes dos integrantes.
